1. 프로젝트 개요 (Project Overview)
1.1. 프로젝트명
대구 빅데이터 활용센터 성과 관리 시스템 (Simple Staff Board)

1.2. 프로젝트 목적
업무 가시성 확보: 담당자별로 산재된 대구시·국비·AI 사업의 진행 현황을 한눈에 파악한다.

신속한 병목 해결: 직관적인 신호등(Traffic Light) UI를 통해 지연 업무를 즉시 식별하고 관리자가 개입한다.

피드백 루프 단축: 불필요한 보고 절차를 줄이고, 시스템 내 원클릭 피드백 기능을 통해 실시간 소통을 강화한다.

1.3. 개발 범위
본 프로젝트는 웹 기반(Web-based) 어플리케이션으로, 다음 두 가지 핵심 뷰(View)를 제공한다.

관리자 모드 (Admin - 센터장):

전체 직원 업무 현황 대시보드 (Masonry Card UI)

프로젝트 유형별(시/국비/AI) 및 상태별 필터링

개별 업무에 대한 피드백 작성 및 전송 기능

이용자 모드 (User - 직원):

본인 할당 업무 조회 (My Task)

업무 상태(진행/지연/완료) 및 진도율 업데이트

관리자 피드백 수신 알림 및 확인

1.4. 기술 스택 (Tech Stack)
최신 웹 표준을 준수하며, 빠른 개발 속도와 안정적인 데이터 처리를 위해 아래 스택을 사용한다.

Frontend: React, Vite, TypeScript

Styling & UI: Tailwind CSS, shadcn/ui

State Management: Zustand (Global), TanStack Query (Server)

Backend & Database: Supabase (Auth, PostgreSQL, Realtime)

AI Integration: Supabase MCP (DB 설계 및 최적화 보조)

1.5. 기본 사항 및 제약 조건
직관성 (Intuitive): 별도의 매뉴얼 없이도 색상(초록/노랑/빨강)만으로 업무 상태를 직관적으로 인지할 수 있어야 한다.

단순성 (Simplicity): 복잡한 통계나 차트는 배제하고, '할 일(To-Do)'과 '상태(Status)' 관리에 집중한다.

반응형 (Responsive): PC 데스크톱 환경을 최우선으로 하되, 모바일/태블릿에서도 현황 조회가 가능해야 한다.

보안 (Security): Supabase RLS(Row Level Security)를 적용하여 직원은 본인의 업무와 관련된 데이터에만 접근하도록 제한한다.

## 2. 이용자 화면 (User View) PRD

### 2.1. 화면 개요
이용자(직원)가 본인에게 할당된 업무를 조회하고, 상태를 업데이트하며, 관리자(센터장)의 피드백을 확인할 수 있는 인터페이스입니다.

### 2.2. 레이아웃 구조

#### 2.2.1. 헤더 영역 (Header)
**위치**: 화면 최상단, 전체 너비

**구성 요소**:
- **제목**: "대구 빅데이터 활용센터 - [사용자명] 책임 업무 관리"
  - 사용자명은 로그인한 사용자의 이름을 동적으로 표시
- **알림 아이콘**: 우측 상단에 벨 아이콘
  - 새로운 피드백이 있을 경우 빨간색 알림 점(notification dot) 표시
  - 클릭 시 피드백 목록 또는 알림 센터로 이동
- **사용자 프로필 아이콘**: 알림 아이콘 옆에 위치
  - 클릭 시 사용자 정보 메뉴 표시 (로그아웃 등)

**피드백 알림 배너** (조건부 표시):
- 새로운 피드백이 도착했을 때만 표시
- 위치: 헤더 제목 바로 아래, 메인 콘텐츠 영역 상단
- 스타일: 빨간색 배경, 흰색 텍스트
- 메시지: "▲ 센터장님으로부터 새로운 피드백이 도착했습니다!"
- 동작: 클릭 시 해당 피드백으로 스크롤 또는 피드백 팝업 표시

#### 2.2.2. 왼쪽 네비게이션 사이드바 (Left Navigation)
**위치**: 화면 왼쪽, 세로 전체 높이

**구성 요소**:
- **내 업무 전체**: 모든 할당된 업무 조회 (기본 선택 상태)
- **오늘 마감**: 오늘 마감일인 업무만 필터링
- **진행 중**: 현재 진행 중인 업무만 필터링
- **완료**: 완료된 업무만 필터링

**상태 표시**:
- 현재 선택된 메뉴는 배경색이 밝은 회색으로 하이라이트
- 호버 시 시각적 피드백 제공

#### 2.2.3. 메인 콘텐츠 영역 (Main Content - Kanban Board)
**위치**: 네비게이션 사이드바 오른쪽, 헤더 아래

**레이아웃**: 3개의 세로 칼럼으로 구성된 칸반 보드

**칼럼 구성**:
1. **내 업무 목록** (To-Do): 아직 시작하지 않은 할당된 업무
2. **진행 중** (In Progress): 현재 진행 중인 업무
3. **완료** (Completed): 완료된 업무

**칼럼 동작**:
- 각 칼럼은 독립적으로 스크롤 가능
- 태스크 카드는 드래그 앤 드롭으로 칼럼 간 이동 가능 (상태 변경)
- 각 칼럼 상단에 해당 상태의 업무 개수 표시

### 2.3. 태스크 카드 (Task Card)

#### 2.3.1. 카드 구조
각 업무는 하나의 카드로 표시되며, 다음 요소를 포함합니다:

**상단 영역**:
- **상태 표시 점**: 카드 제목 왼쪽에 색상 점 표시
  - 🔴 빨간색: 지연 또는 긴급 (마감일 임박 또는 지연)
  - 🟡 노란색: 주의 필요 (마감일이 가까움)
  - 🟢 초록색: 정상 진행 (여유 있음)
- **제목**: 업무 제목
  - 형식: "[프로젝트 유형] 업무명 (마감일)"
  - 프로젝트 유형: [국비], [AI], [대구시] 등
  - 마감일: (~MM/DD) 형식으로 표시

**하단 영역**:
- **상태 업데이트 버튼**: 업무 상태 및 진도율 수정
- **피드백 확인 버튼**: 관리자 피드백이 있을 경우 표시
  - 피드백이 있을 경우 말풍선 아이콘과 함께 표시
  - 클릭 시 피드백 팝업 표시

**시각적 강조**:
- 새로운 피드백이 있는 카드는 빨간색 테두리로 강조 표시
- 호버 시 카드가 약간 상승하는 애니메이션 효과

#### 2.3.2. 카드 상호작용
- **클릭**: 카드 상세 정보 모달 표시 (선택 사항)
- **드래그 앤 드롭**: 다른 칼럼으로 이동하여 상태 변경
- **상태 업데이트 버튼 클릭**: 상태/진도율 수정 모달 표시
- **피드백 확인 버튼 클릭**: 피드백 팝업 표시

### 2.4. 피드백 팝업 (Feedback Modal)

#### 2.4.1. 표시 조건
- 관리자로부터 새로운 피드백이 도착했을 때
- 태스크 카드의 "피드백 확인" 버튼 클릭 시
- 헤더의 피드백 알림 배너 클릭 시

#### 2.4.2. 팝업 구성
**위치**: 해당 태스크 카드 위에 오버레이로 표시

**구성 요소**:
- **제목**: "센터장님 피드백:"
- **내용 영역**: 피드백 텍스트 표시
  - 텍스트는 그대로 표시
  - 이미지 참조가 있을 경우 "<IMAGE 0> 참조" 형식으로 표시
  - 이미지는 별도 모달 또는 확대 뷰로 제공
- **확인 버튼**: 우측 하단에 파란색 "[확인]" 버튼
  - 클릭 시 피드백을 읽음 처리하고 팝업 닫기
  - 읽음 처리 후 해당 태스크 카드의 빨간색 테두리 제거

**스타일**:
- 흰색 배경의 직사각형 팝업
- 그림자 효과로 카드 위에 떠 있는 느낌
- 반응형으로 모바일에서도 적절히 표시

### 2.5. 상태 업데이트 모달

#### 2.5.1. 기능
사용자가 업무의 상태와 진도율을 업데이트할 수 있는 모달입니다.

#### 2.5.2. 구성 요소
- **업무 제목**: 현재 업데이트 중인 업무명 표시
- **상태 선택**: 드롭다운 또는 라디오 버튼
  - 옵션: "할 일", "진행 중", "완료"
- **진도율 입력**: 슬라이더 또는 숫자 입력 필드 (0-100%)
- **메모 입력** (선택 사항): 상태 변경 사유나 추가 설명
- **저장 버튼**: 변경사항 저장
- **취소 버튼**: 변경사항 취소

### 2.6. 필터링 및 정렬

#### 2.6.1. 네비게이션 필터
왼쪽 사이드바의 메뉴 선택에 따라 자동으로 필터링:
- **내 업무 전체**: 모든 상태의 업무 표시
- **오늘 마감**: 마감일이 오늘인 업무만 표시
- **진행 중**: 상태가 "진행 중"인 업무만 표시
- **완료**: 상태가 "완료"인 업무만 표시

#### 2.6.2. 정렬 (선택 사항)
각 칼럼 내에서 업무를 정렬:
- 기본: 마감일 기준 오름차순 (가까운 순)
- 대체: 프로젝트 유형별 그룹화

### 2.7. 반응형 디자인

#### 2.7.1. 데스크톱 (기본)
- 전체 레이아웃 유지
- 3개 칼럼 가로 배치
- 사이드바 항상 표시

#### 2.7.2. 태블릿
- 사이드바는 햄버거 메뉴로 전환 가능
- 칼럼 너비 조정 (최소 너비 유지)
- 태스크 카드 크기 조정

#### 2.7.3. 모바일
- 사이드바는 햄버거 메뉴로 숨김
- 칼럼은 가로 스크롤 또는 탭 전환 방식
- 태스크 카드 최적화 (텍스트 크기, 간격 조정)

### 2.8. 실시간 업데이트

#### 2.8.1. Supabase Realtime 활용
- 새로운 피드백 도착 시 실시간 알림
- 업무 상태 변경 시 즉시 반영
- 다른 기기에서의 변경사항 동기화

#### 2.8.2. 알림 표시
- 새 피드백 도착: 헤더 알림 아이콘에 빨간 점 표시 + 배너 표시
- 상태 변경 완료: 토스트 메시지로 성공 알림

### 2.9. 접근성 (Accessibility)

#### 2.9.1. 키보드 네비게이션
- Tab 키로 모든 인터랙티브 요소 순회
- Enter/Space로 버튼 활성화
- Escape로 모달/팝업 닫기

#### 2.9.2. 스크린 리더 지원
- 모든 버튼과 링크에 적절한 aria-label 제공
- 상태 표시 점에 색상 외 텍스트 설명 추가
- 폼 요소에 적절한 레이블 연결

### 2.10. 성능 요구사항

#### 2.10.1. 로딩 시간
- 초기 페이지 로드: 2초 이내
- 필터 변경: 500ms 이내
- 상태 업데이트: 1초 이내

#### 2.10.2. 최적화
- 태스크 카드 가상 스크롤링 (많은 업무가 있을 경우)
- 이미지 지연 로딩
- API 호출 최소화 (캐싱 활용)

### 2.11. 에러 처리

#### 2.11.1. 네트워크 오류
- 연결 실패 시 재시도 버튼 제공
- 오프라인 상태 표시

#### 2.11.2. 데이터 오류
- 업무 로드 실패 시 에러 메시지 표시
- 상태 업데이트 실패 시 롤백 및 알림

### 2.12. 사용자 경험 (UX) 고려사항

#### 2.12.1. 시각적 피드백
- 버튼 호버 효과
- 로딩 스피너 (데이터 로딩 중)
- 성공/실패 토스트 메시지

#### 2.12.2. 직관성
- 색상 코딩으로 상태 즉시 인지 (신호등 시스템)
- 아이콘과 텍스트의 명확한 조합
- 일관된 인터랙션 패턴

## 3. 관리자 화면 (Admin View) PRD

### 3.1. 화면 개요
관리자(센터장)가 전체 직원의 업무 현황을 한눈에 파악하고, 개별 업무에 대해 피드백을 작성하여 전송할 수 있는 대시보드 인터페이스입니다.

### 3.2. 레이아웃 구조

#### 3.2.1. 헤더 영역 (Header)
**위치**: 화면 최상단, 전체 너비

**구성 요소**:
- **제목**: "대구 빅데이터 활용센터 - 관리자 대시보드"
  - 관리자 전용 화면임을 명확히 표시
- **알림 아이콘**: 우측 상단에 벨 아이콘
  - 새로운 알림이 있을 경우 빨간색 알림 점(notification dot) 표시
  - 클릭 시 알림 목록 또는 알림 센터로 이동
- **사용자 프로필 아이콘**: 알림 아이콘 옆에 위치
  - 클릭 시 사용자 정보 메뉴 표시 (로그아웃, 설정 등)

#### 3.2.2. 왼쪽 네비게이션 사이드바 (Left Navigation)
**위치**: 화면 왼쪽, 세로 전체 높이

**구성 요소**:
- **전체 직원**: 모든 직원의 업무 현황 조회 (기본 선택 상태)
- **피드백 필요**: 피드백이 필요한 업무만 필터링
  - 직원이 업데이트를 요청했거나, 지연된 업무 등
- **[대구시 사업]**: 대구시 사업 관련 업무만 필터링
- **[국비 데이터 사업]**: 국비 데이터 사업 관련 업무만 필터링
- **[AI 사업]**: AI 사업 관련 업무만 필터링

**상태 표시**:
- 현재 선택된 메뉴는 배경색이 밝은 회색으로 하이라이트
- 호버 시 시각적 피드백 제공
- 프로젝트 유형별 필터는 해당 유형의 업무가 있는 직원만 표시

#### 3.2.3. 메인 콘텐츠 영역 (Main Content - Employee Work Status Board)
**위치**: 네비게이션 사이드바 오른쪽, 헤더 아래

**레이아웃**: 직원별 세로 칼럼으로 구성된 Masonry Card UI

**제목**: "직원별 업무 현황 보드" (Employee Work Status Board)

**칼럼 구성**:
- 각 직원마다 하나의 세로 칼럼 생성
- 칼럼은 가로로 나열되며, 필요 시 가로 스크롤 가능
- 각 칼럼은 독립적으로 세로 스크롤 가능

**칼럼 헤더**:
- **직원 정보**: 직원 이름 및 직급 표시
  - 형식: "[이름] [직급]" (예: "김 책임 (팀장)")
- **업무 통계**: 색상 점으로 업무 개수 표시
  - 🟢 초록색 점: 정상 진행 중인 업무 개수
  - 🔴 빨간색 점: 지연 또는 긴급 업무 개수
  - 형식: "🟢[개수] 🔴[개수]" (예: "🟢3 🔴1")
  - 노란색 업무가 있을 경우: "🟡[개수]" 추가 표시

**칼럼 동작**:
- 각 칼럼 내부는 해당 직원의 모든 업무 카드가 세로로 배치
- 칼럼 너비는 고정 또는 가변 가능 (최소 너비 유지)
- 많은 직원이 있을 경우 가로 스크롤 제공

### 3.3. 태스크 카드 (Task Card)

#### 3.3.1. 카드 구조
각 업무는 하나의 카드로 표시되며, 다음 요소를 포함합니다:

**상단 영역**:
- **상태 표시 점**: 카드 제목 왼쪽에 색상 점 표시
  - 🔴 빨간색: 지연 또는 긴급 (마감일 임박 또는 지연)
  - 🟡 노란색: 주의 필요 (마감일이 가까움)
  - 🟢 초록색: 정상 진행 (여유 있음)
- **제목**: 업무 제목
  - 형식: "[프로젝트 유형] 업무명 (마감일)"
  - 프로젝트 유형: [국비], [AI], [대구시] 등
  - 마감일: (~MM/DD) 형식으로 표시

**하단 영역**:
- **피드백 버튼**: 말풍선 아이콘과 함께 "피드백" 텍스트
  - 모든 카드에 표시 (피드백 작성/확인용)
  - 클릭 시 피드백 작성 팝업 표시
  - 이미 피드백이 있는 경우 시각적 표시 (예: 아이콘 색상 변경)

**시각적 강조**:
- 빨간색 점이 있는 카드는 더 두드러지게 표시
- 호버 시 카드가 약간 상승하는 애니메이션 효과
- 피드백이 필요한 카드는 테두리 강조

#### 3.3.2. 카드 상호작용
- **클릭**: 카드 상세 정보 모달 표시 (선택 사항)
- **피드백 버튼 클릭**: 피드백 작성 팝업 표시

### 3.4. 피드백 작성 팝업 (Feedback Modal)

#### 3.4.1. 표시 조건
- 태스크 카드의 "피드백" 버튼 클릭 시
- "피드백 필요" 필터에서 해당 업무 클릭 시

#### 3.4.2. 팝업 구성
**위치**: 화면 중앙 또는 해당 태스크 카드 근처에 오버레이로 표시

**구성 요소**:
- **업무명**: "업무명: [업무 제목]"
  - 현재 피드백을 작성하는 업무의 제목 표시
- **직원 업데이트**: "직원명: '[직원이 작성한 최신 업데이트 내용]'"
  - 해당 직원이 마지막으로 작성한 상태 업데이트 또는 메모 표시
  - 업데이트가 없는 경우 "업데이트 없음" 표시
- **센터장 피드백 입력창**: 
  - 라벨: "센터장 피드백:"
  - 다중 라인 텍스트 입력 필드
  - 플레이스홀더: "피드백을 입력하세요..."
  - 이미지 첨부 기능 (선택 사항)
- **전송 버튼**: 우측 하단에 파란색 "[전송]" 버튼
  - 클릭 시 피드백을 해당 직원에게 전송
  - 전송 성공 시 팝업 닫기 및 성공 메시지 표시
  - 전송 실패 시 에러 메시지 표시

**스타일**:
- 흰색 배경의 직사각형 팝업
- 그림자 효과로 화면 위에 떠 있는 느낌
- 반응형으로 모바일에서도 적절히 표시
- 입력창은 충분한 높이 제공 (최소 3-4줄)

#### 3.4.3. 피드백 작성 기능
- **텍스트 입력**: 일반 텍스트 입력 지원
- **이미지 첨부** (선택 사항): 
  - 이미지 업로드 버튼 제공
  - 업로드된 이미지는 미리보기 표시
  - 이미지 참조 시 "<IMAGE 0> 참조" 형식으로 텍스트에 자동 삽입
- **이전 피드백 확인**: 
  - 이전에 작성한 피드백이 있는 경우 표시
  - 피드백 히스토리 보기 기능 (선택 사항)

### 3.5. 필터링 및 정렬

#### 3.5.1. 네비게이션 필터
왼쪽 사이드바의 메뉴 선택에 따라 자동으로 필터링:
- **전체 직원**: 모든 직원의 모든 업무 표시
- **피드백 필요**: 
  - 직원이 업데이트를 요청한 업무
  - 지연된 업무 (빨간색 점)
  - 마감일이 임박한 업무 (노란색 점)
- **[대구시 사업]**: 대구시 사업 관련 업무만 표시
- **[국비 데이터 사업]**: 국비 데이터 사업 관련 업무만 표시
- **[AI 사업]**: AI 사업 관련 업무만 표시

#### 3.5.2. 정렬 (선택 사항)
- **직원 정렬**: 
  - 기본: 이름 가나다순 또는 직급순
  - 대체: 업무 개수순 (많은 순/적은 순)
- **업무 정렬**: 각 칼럼 내에서
  - 기본: 마감일 기준 오름차순 (가까운 순)
  - 대체: 상태별 그룹화 (빨강 → 노랑 → 초록)

### 3.6. 업무 현황 요약 (선택 사항)

#### 3.6.1. 대시보드 통계
헤더 아래 또는 사이드바 상단에 간단한 통계 표시:
- 전체 업무 수
- 지연 업무 수 (빨간색)
- 오늘 마감 업무 수
- 피드백 대기 중인 업무 수

#### 3.6.2. 시각적 인디케이터
- 각 직원 칼럼의 통계 점(🟢🔴)으로 한눈에 파악
- 지연 업무가 많은 직원의 칼럼은 시각적으로 강조

### 3.7. 반응형 디자인

#### 3.7.1. 데스크톱 (기본)
- 전체 레이아웃 유지
- 직원 칼럼 가로 배치 (필요 시 스크롤)
- 사이드바 항상 표시
- 최소 3-4명의 직원 칼럼을 동시에 표시 가능

#### 3.7.2. 태블릿
- 사이드바는 햄버거 메뉴로 전환 가능
- 직원 칼럼 너비 조정 (최소 너비 유지)
- 동시 표시 칼럼 수 감소 (2-3개)
- 태스크 카드 크기 조정

#### 3.7.3. 모바일
- 사이드바는 햄버거 메뉴로 숨김
- 직원 칼럼은 가로 스크롤 또는 탭 전환 방식
- 한 번에 1-2개의 직원 칼럼만 표시
- 태스크 카드 최적화 (텍스트 크기, 간격 조정)

### 3.8. 실시간 업데이트

#### 3.8.1. Supabase Realtime 활용
- 직원의 업무 상태 변경 시 실시간 반영
- 새로운 업무 할당 시 즉시 표시
- 피드백 전송 완료 시 직원에게 실시간 알림
- 다른 기기에서의 변경사항 동기화

#### 3.8.2. 알림 표시
- 새로운 업데이트: 해당 업무 카드에 시각적 표시
- 피드백 전송 완료: 토스트 메시지로 성공 알림
- 지연 업무 발생: 해당 칼럼 헤더의 빨간색 점 업데이트

### 3.9. 접근성 (Accessibility)

#### 3.9.1. 키보드 네비게이션
- Tab 키로 모든 인터랙티브 요소 순회
- Enter/Space로 버튼 활성화
- Escape로 모달/팝업 닫기
- 화살표 키로 칼럼 간 이동 (선택 사항)

#### 3.9.2. 스크린 리더 지원
- 모든 버튼과 링크에 적절한 aria-label 제공
- 상태 표시 점에 색상 외 텍스트 설명 추가
- 직원 칼럼에 적절한 레이블 제공
- 폼 요소에 적절한 레이블 연결

### 3.10. 성능 요구사항

#### 3.10.1. 로딩 시간
- 초기 페이지 로드: 3초 이내 (모든 직원 데이터 포함)
- 필터 변경: 1초 이내
- 피드백 전송: 1초 이내

#### 3.10.2. 최적화
- 직원 칼럼 가상 스크롤링 (많은 직원이 있을 경우)
- 태스크 카드 지연 로딩 (보이는 영역만 먼저 로드)
- 이미지 지연 로딩
- API 호출 최소화 (캐싱 활용)
- 실시간 업데이트는 필요한 부분만 업데이트

### 3.11. 에러 처리

#### 3.11.1. 네트워크 오류
- 연결 실패 시 재시도 버튼 제공
- 오프라인 상태 표시
- 피드백 전송 실패 시 재시도 옵션

#### 3.11.2. 데이터 오류
- 업무 로드 실패 시 에러 메시지 표시
- 피드백 전송 실패 시 롤백 및 알림
- 직원 데이터 없음 시 안내 메시지

### 3.12. 사용자 경험 (UX) 고려사항

#### 3.12.1. 시각적 피드백
- 버튼 호버 효과
- 로딩 스피너 (데이터 로딩 중)
- 성공/실패 토스트 메시지
- 피드백 전송 중 로딩 표시

#### 3.12.2. 직관성
- 색상 코딩으로 상태 즉시 인지 (신호등 시스템)
- 직원별 칼럼으로 업무 분산 명확히 표시
- 통계 점으로 한눈에 파악 가능
- 아이콘과 텍스트의 명확한 조합
- 일관된 인터랙션 패턴

#### 3.12.3. 효율성
- 빠른 피드백 작성 (원클릭 접근)
- 필터링으로 필요한 정보만 표시
- 키보드 단축키 지원 (선택 사항)
- 자주 사용하는 필터 즐겨찾기 (선택 사항)
